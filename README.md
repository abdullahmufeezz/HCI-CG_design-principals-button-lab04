# HCI Principles Demo – Button Interaction Lab

## 📌 Overview
This project demonstrates key **Human-Computer Interaction (HCI) principles** using a simple interactive button interface built with **HTML, CSS, and JavaScript**.
I have made a change in HTML file where i thought i would make it more better according to the Human computer Interaction Rules.
The system allows users to click a button up to **3 times**, while visually and functionally applying HCI concepts to improve usability and user experience.

---

##  HCI Principles Used

### 1. Affordance
**Definition:**  
Indicates what actions the user can perform.

**Implementation:**  
- The button is styled with a blue border and pointer cursor.
- It visually suggests that it is clickable.

---

### 2. Signifier
**Definition:**  
Provides cues about how to use an interface.

**Implementation:**  
- Button text: `Process Data (Click me!)`
- Instruction message: *"You can click the button up to 3 times"*
- Disabled button changes color (purple)

---

### 3. Feedback
**Definition:**  
System response after user interaction.

**Implementation:**  
- Button turns yellow briefly when clicked
- Status message updates with number of clicks
- Final message when limit is reached

---

### 4. Constraint
**Definition:**  
Restricts user actions to prevent errors.

**Implementation:**  
- User can only click the button **3 times**
- Button becomes disabled after reaching limit

---

### 5. Mapping
**Definition:**  
Clear relationship between action and result.

**Implementation:**  
- Clicking the button updates the status box
- The output directly reflects user interaction

---

### 6. Visibility
**Definition:**  
Important elements and information are always visible.

**Implementation:**  
- Instructions are shown at the top
- Status box is always displayed
- Button state changes are clearly visible

---

### 7. Consistency
**Definition:**  
Uniform design and behavior across the interface.

**Implementation:**  
- Color coding:
  - Blue → Action
  - Green → Status
  - Yellow → Feedback
  - Red → Error/Limit
  - Purple → Disabled state

---

## ✅ Conclusion
This project successfully demonstrates how **HCI principles improve usability** by making the interface:
- Clear
- Interactive
- User-friendly
- Error-resistant
